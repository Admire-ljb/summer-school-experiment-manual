Experiment 11: Integrated Practice

Confirm the Group URI Before Running

Python scripts in this experiment read the complete radio URI assigned in Experiment 3 from the CFLIB_URI environment variable. Do not copy the group address into every code file. In each new terminal, check it first:

echo "$CFLIB_URI"

The output must exactly match the complete URI on the group label, including the interface, channel, data rate, and address. If it is empty or incorrect, repeat the “Save the complete URI in the virtual machine” step in Experiment 3 before running the script.

1. Stage integrated practice instructions

The comprehensive practice at this stage is used to test students' mastery of the basic control of the Crazyflie drone, flight function calls and task process organization. As long as the drone can complete specified flight actions through programming, or basically complete the main actions and explain the implementation ideas, this practice can be regarded as completed. Encourage everyone to collaborate in debugging in groups and pay attention to on-site order and flight safety.

2. Practice tasks

There are two tasks set up for this practice. Completing any one of them can be regarded as the completion of the comprehensive practice in the stage.

In addition, let me explain in advance that the drone sensor used by the professor so far is flow deck, which mainly uses optical flow sensors to achieve speed positioning. If the ground is smooth and reflective, it will cause inaccurate positioning and the drone's position will easily drift. The best way to solve this problem is to fly on a non-reflective ground. For this purpose, a carpet has been specially laid for everyone. You can test fly on the carpet and show it to the teacher when you are ready. Pay attention to the orderliness of the entire process and flight safety.

Task 1: Pattern flight

First, the drone automatically lifts off to 0.3m. Let it fly a little higher to a height of 0.4m. Stabilize the attitude. Then, let the drone stay still. First, rotate 360 ​​degrees to the right on the spot, and then rotate 360 ​​degrees to the left. After stabilizing the attitude of the drone, let the drone fly a figure-8 trajectory and then land.

Think about what functions this process will call to control the flight of the drone. In order to ensure flight safety, the radius and speed of the figure-8 flight trajectory can be set smaller so that the flight attitude can be clearly seen. After the flight is completed, display the flight code to complete the task.

Task 2: Cruise flight

The flight map is as shown in the figure below. There are three obstacles ABC, shown by the blue squares. The placement distance of these three obstacles has been marked in the figure. The drone needs to take off from any position in the lower left corner of obstacle A. The forward direction of the drone when taking off is not restricted. Follow the flight path diagram shown by the purple arrow, bypass obstacles B and C, and finally land in the triangular area surrounded by three obstacles ABC. The drone landing on the boundary line is considered successful.

Please note that the purple trajectory is just a hint. The drone does not have to fly an arc trajectory. The same effect can be achieved by going straight, turning around, and then going straight again.
