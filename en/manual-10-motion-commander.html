1. Deep understanding of motion commander

First, let's review the Motion Commander class code we used before.

Here you can refer to the content in test.py we used in the fourth lesson. This python code implements some simple actions to control the drone.

import logging
import time
import cflib.crtp
from cflib.crazyflie.syncCrazyflie import SyncCrazyflie
from cflib.positioning.motion_commander import MotionCommander
URI = 'radio://0/80/250K'  #Change to your cf2's URI
# Only output errors from the logging framework
logging.basicConfig(level=logging.ERROR)
if __name__ == '__main__':
# Initialize the low-level drivers (don't list the debug drivers)
cflib.crtp.init_drivers(enable_debug_driver=False)
with SyncCrazyflie(URI) as scf:
# We take off when the commander is created
with MotionCommander(scf) as mc:
print('Taking off!')
time.sleep(1)
# There is a set of functions that move a specific distance
# We can move in all directions
print('Moving forward 0.5m')
mc.forward(0.5)
# Wait a bit
time.sleep(1)
print('Moving up 0.2m')
mc.up(0.2)
# Wait a bit
time.sleep(1)
print('Doing a 270deg circle');
mc.circle_right(0.5, velocity=0.5, angle_degrees=270)
print('Moving down 0.2m')
mc.down(0.2)
# Wait a bit
time.sleep(1)
print('Rolling left 0.2m at 0.6m/s')
mc.left(0.2, velocity=0.6)
# Wait a bit
time.sleep(1)
print('Moving forward 0.5m')
mc.forward(0.5)
# We land when the MotionCommander goes out of scope
print('Landing!')

Let's review the relevant content we have learned. Let's focus on the core part of this code. Starting from line 17, we create a motion commander entity class mc. After this entity mc is created, it is responsible for controlling the movement of the drone. The moment it is created, it will automatically execute the drone take-off program, which takes off to a height of 0.3m by default. On line 18, we output taking off in the terminal, indicating that the drone has taken off, and stop the entire program for 1 second to allow the drone to stabilize its flight status. In the following code, you can see that we call a sub-function forward of the mc class to make the drone move forward 0.5m and then stop. Note that the hardware that this code is adapted to now is its own gyroscope and flowdeck module. The speed and altitude of the drone are provided by flowdeck, but its accuracy is limited, and the deviation may sometimes be slightly larger. In this case, ensure that the drone itself is in a normal state, and the parameters passed to the function can be adjusted to allow the drone to fly to the ideal position. For example, if 0.5 is entered but the actual drone travels 0.1m more, the parameters passed in can be reduced so that the drone can actually go to the appropriate position.

Looking further down, all the places where the time.sleep(1) function is used are to keep the drone in its current state for 1 second to stabilize its attitude. On line 17, the mc class calls the sub-function of up. This statement causes the height of the drone to rise by 0.2m. It can be seen from here that the functions of many sub-functions of this class can be seen from the naming. Let’s look at another complicated one. The circle_right sub-function on line 31 passes in three parameters. The function of this sub-function is to make the drone fly to the right starting from the original position and flying an arc trajectory. The first parameter passed in, 0.5, represents the radius of the circle of 0.5m, the second parameter velocity represents the speed of 0.5m/s, and the third parameter angle_degrees represents the degree of the rotation arc. Assigning a value of 360 will cause the drone to complete a full circle. The subsequent code can also infer its effect from its naming.

So how to rigorously understand the functions of the sub-functions of the motion commander class and how to pass in the parameters? We can click to view the official open source website. Of course, the cflib library has been installed in the virtual machine, and the source code can also be viewed. It is convenient for everyone to view it. The key parts of the source code are directly posted for everyone, and they are also placed in the folder of this lesson.

Next, I will introduce to you the commonly used sub-functions of the motion commander class. You can call them yourself and try to make the parameters as small as possible to ensure safety.

Example of calling method: mc.left(0.2,velocity=0.6)

That is, the drone moves 0.2m to the right with a speed of 0.6m/s

Similarly, the following functions are used in the same way. Note that the velocity parameter does not need to be set, and the default speed of 0.1m/s will be used. Please refer to the previous usage of test.py.

Next is another type of sub-function, the turn_left sub-function can control the forward direction of the drone to turn left.

Usage example: mc.turn_left(90,rate=45) means that the drone turns 90 degrees to the left, and the turning angular speed is 45 degrees per second. Similarly, the default angular speed can be used without passing in the speed.

Similarly, there are sub-functions of turn_right. You can test whether the final effects of turn_right(180) and turn_left(180) are the same.

The definition and implementation of the sub-function circle_left for turning circles to the left is as follows. There is also a sub-function method for circle_right.

Finally, I will introduce you to the usage of the move_distance sub-function. You can see that this function is used in the definition of many previous functions. This function can pass in a total of four values. The first three values ​​represent the displacement distance of the drone on the xyz axis of the three-dimensional space. The last value represents the speed. Using this function, the drone can move in any direction in the three-dimensional space.

The total area is a 1.5*1.5m rectangle. The vertical projection of the drone must not fall into the red area during operation, nor can it exceed the rectangular area.
