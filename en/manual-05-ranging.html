Experiment 5: Advanced Ranging Experiment

Confirm the Group URI Before Running

Python scripts in this experiment read the complete radio URI assigned in Experiment 3 from the CFLIB_URI environment variable. Do not copy the group address into every code file. In each new terminal, check it first:

echo "$CFLIB_URI"

The output must exactly match the complete URI on the group label, including the interface, channel, data rate, and address. If it is empty or incorrect, repeat the “Save the complete URI in the virtual machine” step in Experiment 3 before running the script.

Experiment 1: Take off and proceed, obstacle landing detected

Download the reviewed script: multiranger_stop.py

#!/usr/bin/env python3
"""Fly forward slowly and land before the front obstacle."""

import threading
import time

import cflib.crtp
from cflib.crazyflie import Crazyflie
from cflib.crazyflie.syncCrazyflie import SyncCrazyflie
from cflib.positioning.motion_commander import MotionCommander
from cflib.utils import uri_helper
from cflib.utils.multiranger import Multiranger


URI = uri_helper.uri_from_env(default="")
FRONT_STOP_DISTANCE_M = 0.28
UP_STOP_DISTANCE_M = 0.15
FORWARD_SPEED_MPS = 0.10
MAX_FLIGHT_TIME_S = 20.0


def request_arming(scf, armed):
    for service_name in ("platform", "supervisor"):
        service = getattr(scf.cf, service_name, None)
        request = getattr(service, "send_arming_request", None)
        if request is not None:
            request(armed)
            return
    raise RuntimeError("This cflib version has no arming service.")


def wait_for_decks(scf):
    attached = {
        "bcFlow2": threading.Event(),
        "bcMultiranger": threading.Event(),
    }

    def callback(name, value):
        deck_name = name.rsplit(".", 1)[-1]
        if deck_name in attached and int(value):
            attached[deck_name].set()

    for deck_name in attached:
        scf.cf.param.add_update_callback(
            group="deck", name=deck_name, cb=callback
        )
    time.sleep(1.0)
    return all(event.wait(timeout=4.0) for event in attached.values())


def main():
    if not URI:
        raise RuntimeError("CFLIB_URI is empty; configure the group URI first.")
    cflib.crtp.init_drivers()
    with SyncCrazyflie(URI, cf=Crazyflie(rw_cache="./cache")) as scf:
        if not wait_for_decks(scf):
            raise RuntimeError(
                "Flow deck or Multi-ranger deck was not detected; takeoff cancelled."
            )
        armed = False
        try:
            request_arming(scf, True)
            armed = True
            time.sleep(1.0)
            with MotionCommander(scf, default_height=0.35) as commander:
                with Multiranger(scf) as multiranger:
                    deadline = time.monotonic() + MAX_FLIGHT_TIME_S
                    while time.monotonic() < deadline:
                        front = multiranger.front
                        up = multiranger.up
                        should_stop = (
                            up is not None and up < UP_STOP_DISTANCE_M
                        ) or (
                            front is not None and front < FRONT_STOP_DISTANCE_M
                        )
                        if should_stop:
                            commander.stop()
                            break
                        commander.start_forward(FORWARD_SPEED_MPS)
                        time.sleep(0.1)
                    commander.stop()
        finally:
            if armed:
                request_arming(scf, False)


if __name__ == "__main__":
    main()

This code implements a simple function. The drone takes off, stabilizes its attitude for 1 second and flies forward at a speed of 0.3m per second. Until an obstacle is detected within 0.4m in front of the drone, it will stop moving forward and land.

Program execution flow

Start the program → Connect the drone → Prepare to unlock

Start flying → fly forward at a speed of 0.3 meters/second

Continuous detection → Checks ahead and above every 0.1 seconds

Encounter an obstacle → Stop flying if the distance between the front and the top is less than 40 cm

End task → print "Demo terminated!"

Students need to understand some necessary statements in the code. First, for the is_close function, this function defines a variable value of 0.3. This function will return whether the multiranger sensor detects obstacles within a certain distance in a certain direction. The calling method can refer to the location where the function is called in the code and the parameters passed in. The distance 0.3 defined by MIN_DISTANCE means that the function will return a certain value after the multiranger sensor detects an obstacle within 0.3m in a certain direction.

At the same time, for the while loop in the main program, a flag called keep_flying is always being judged. When the flag is true, the while loop will continue to execute the following statements (until the time.sleep (0.1) statement). When the while statement ends, that is, the keep_flying flag is set to false, the while loop will be jumped out, and the code after the while loop will be executed, which is the print statement. At the same time, the motioncommander class will be automatically destroyed after being executed sequentially, and the drone will automatically land. That's how this code works.

Experiment 3: "Bounce" in a box

Run the complete code below (i.e. Bouncing_box.py):

Note that before running this code, try to place the drone in the center of the operating site. This code mainly calls the function of the flowdeck module to achieve visual navigation and positioning, and does not use the function of multiranger. Let the drone "bounce" in a square area, that is, every time it comes into contact with the invisible square boundary, the speed of the drone in the x and y directions changes, achieving a "bounce back" effect.

Because the flowdeck function is used, the best effect is only run on a non-reflective site. It will also bounce infinitely on this invisible square area until the program is manually stopped. BOX_LIMIT = 0.3 sets the size of the square area. Students can modify this value appropriately. Note that it is best to place it in the middle of the field when taking off.

Download the reviewed script: flow_box_bounce.py

#!/usr/bin/env python3
"""Bounded Flow-deck position exercise with stale-log and time limits."""

import threading
import time

import cflib.crtp
from cflib.crazyflie import Crazyflie
from cflib.crazyflie.log import LogConfig
from cflib.crazyflie.syncCrazyflie import SyncCrazyflie
from cflib.positioning.motion_commander import MotionCommander
from cflib.utils import uri_helper


URI = uri_helper.uri_from_env(default="")
DEFAULT_HEIGHT_M = 0.35
BOX_LIMIT_M = 0.35
MAX_VELOCITY_MPS = 0.15
MAX_FLIGHT_TIME_S = 45.0
LOG_STALE_TIMEOUT_S = 0.60

flow_deck_attached = threading.Event()
position_lock = threading.Lock()
position_estimate = [0.0, 0.0]
last_position_update = 0.0


def request_arming(scf, armed):
    for service_name in ("platform", "supervisor"):
        service = getattr(scf.cf, service_name, None)
        request = getattr(service, "send_arming_request", None)
        if request is not None:
            request(armed)
            return
    raise RuntimeError("This cflib version has no arming service.")


def flow_deck_callback(name, value):
    if int(value):
        flow_deck_attached.set()


def position_callback(timestamp, data, log_config):
    global last_position_update
    with position_lock:
        position_estimate[0] = float(data["stateEstimate.x"])
        position_estimate[1] = float(data["stateEstimate.y"])
        last_position_update = time.monotonic()


def move_box_limit(scf):
    body_x = MAX_VELOCITY_MPS
    body_y = MAX_VELOCITY_MPS * 0.5
    deadline = time.monotonic() + MAX_FLIGHT_TIME_S

    with MotionCommander(scf, default_height=DEFAULT_HEIGHT_M) as commander:
        while time.monotonic() < deadline:
            with position_lock:
                x, y = position_estimate
                update_age = time.monotonic() - last_position_update
            if update_age > LOG_STALE_TIMEOUT_S:
                commander.stop()
                raise RuntimeError("Position log is stale; landing.")
            if x >= BOX_LIMIT_M:
                body_x = -MAX_VELOCITY_MPS
            elif x <= -BOX_LIMIT_M:
                body_x = MAX_VELOCITY_MPS
            if y >= BOX_LIMIT_M:
                body_y = -MAX_VELOCITY_MPS
            elif y <= -BOX_LIMIT_M:
                body_y = MAX_VELOCITY_MPS
            commander.start_linear_motion(body_x, body_y, 0.0)
            time.sleep(0.1)
        commander.stop()


def main():
    if not URI:
        raise RuntimeError("CFLIB_URI is empty; configure the group URI first.")
    cflib.crtp.init_drivers()
    with SyncCrazyflie(URI, cf=Crazyflie(rw_cache="./cache")) as scf:
        scf.cf.param.add_update_callback(
            group="deck", name="bcFlow2", cb=flow_deck_callback
        )
        time.sleep(1.0)
        if not flow_deck_attached.wait(timeout=5.0):
            raise RuntimeError("Flow deck was not detected; takeoff cancelled.")

        position_log = LogConfig(name="Position", period_in_ms=1000)
        position_log.add_variable("stateEstimate.x", "float")
        position_log.add_variable("stateEstimate.y", "float")
        scf.cf.log.add_config(position_log)
        position_log.data_received_cb.add_callback(position_callback)
        log_started = False
        armed = False
        try:
            position_log.start()
            log_started = True
            if not wait_for_first_position(timeout=2.0):
                raise RuntimeError("No position log data; takeoff cancelled.")
            request_arming(scf, True)
            armed = True
            time.sleep(1.0)
            move_box_limit(scf)
        finally:
            if log_started:
                position_log.stop()
            if armed:
                request_arming(scf, False)


def wait_for_first_position(timeout):
    deadline = time.monotonic() + timeout
    while time.monotonic() < deadline:
        with position_lock:
            if last_position_update > 0.0:
                return True
        time.sleep(0.05)
    return False


if __name__ == "__main__":
    main()

Experiment 4 (optional extension): Wall-following Flight

Single code source: the official Bitcraze example

This experiment uses the wall-following example from Bitcraze's official crazyflie-demos repository. No separate classroom wall-following implementation is provided. The multiranger_wall_following.py entry point and its wall_following.py state-machine dependency must remain together in the official example directory.

  • Official repository: https://github.com/bitcraze/crazyflie-demos/tree/main
  • Wall-following example: https://github.com/bitcraze/crazyflie-demos/tree/main/demos/scripts/cflib/multiranger/multiranger_wall_following

Clone the official repository once and enter the example directory. If crazyflie-demos is already present, enter the existing directory instead of making another copy.

cd ~/workspace
git clone https://github.com/bitcraze/crazyflie-demos.git
cd crazyflie-demos/demos/scripts/cflib/multiranger/multiranger_wall_following
echo "$CFLIB_URI"

In the official multiranger_wall_following.py, change only the WallFollowing(...) initialization to the following recommended settings. Because reference_distance_from_wall + ranger_value_buffer = 0.24 m, corner handling starts when the front range falls below approximately 0.24 m.

wall_following = WallFollowing(
    angle_value_buffer=0.1,
    reference_distance_from_wall=0.18,
    max_forward_speed=0.08,
    max_turn_rate=0.35,
    ranger_value_buffer=0.06,
    range_lost_threshold=0.30,
    init_state=WallFollowing.StateWallFollowing.FORWARD)

To follow a left-hand wall, use the official RIGHT + left_range combination. To follow a right-hand wall, use LEFT + right_range. The enum names the turning direction, not the side on which the wall is located. Keep only the pair required for the run.

# Follow a wall on the left
wall_following_direction = WallFollowing.WallFollowingDirection.RIGHT
side_range = left_range

# Follow a wall on the right
wall_following_direction = WallFollowing.WallFollowingDirection.LEFT
side_range = right_range

After verifying that CFLIB_URI contains the complete URI assigned to the group, run the official entry point from this directory:

python3 multiranger_wall_following.py

Operating conditions and first test

The official example is intended for continuous straight walls with corners. Panel seams, low obstacles, and large openings may briefly remove a single-point range return. Cover obvious seams first, verify the selected wall side, turning direction, and upper-sensor stop gesture beside one straight wall at low speed, and only then enter the 40-50 cm corridor. One team member must continuously observe the aircraft and be ready to stop the run.

This code mainly implements the function of the drone walking along the wall to achieve the purpose of exploring along the wall. Students can use this to experience the concept of autonomous exploration by the drone and pave the way for subsequent courses.
