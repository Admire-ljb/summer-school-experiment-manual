Experiment 6: Complex-map Flight and Mapping

Confirm the Group URI Before Running

Python scripts in this experiment read the complete radio URI assigned in Experiment 3 from the CFLIB_URI environment variable. Do not copy the group address into every code file. In each new terminal, check it first:

echo "$CFLIB_URI"

The output must exactly match the complete URI on the group label, including the interface, channel, data rate, and address. If it is empty or incorrect, repeat the “Save the complete URI in the virtual machine” step in Experiment 3 before running the script.

1. Experimental goals

Continue to familiarize yourself with the programming methods related to the multiranger sensor in the cflib library.

Use multiranger sensors to achieve more interesting flight missions

2. Experimental preparation

The virtual machine system of the client connected to the drone has been configured previously. And you need to have the Linux system operating experience accumulated in the previous experiments. A little more basic python programming is needed.

Hardware equipment needed in this experiment:

  • Crazyflie 2.0
  • Crazyradio PA
  • Flow deck
  • Multiranger deck

3. Experimental steps

This experiment mainly runs Python scripts. Open the Bitcraze cflib User Guides directly:

https://www.bitcraze.io/documentation/repository/crazyflie-lib-python/master/user-guides/

If you encounter difficulties during the experiment, open the URL listed above for further reference. Some operations and knowledge points are also mentioned in the previous experiment manual; this part is not repeated here.

Experiment 1: walk follow the wall

This experiment is an expanded experiment of the previous class. Now the parameters that can be adjusted in the code are given. Students can modify the parameters to test the operation effect.

Official demo repository: Multiranger Wall Following

wall_following.py and multiranger_wall_following.py come from Bitcraze's official crazyflie-demos repository. Use the files in the official repository as the source of record; this manual lists the source links and classroom usage steps.

  • Repository: https://github.com/bitcraze/crazyflie-demos/tree/main
  • Demo directory: demos/scripts/cflib/multiranger/multiranger_wall_following
  • wall_following.py: official file link
  • multiranger_wall_following.py: official file link

Recommended usage in the virtual machine:

cd ~/workspace
git clone https://github.com/bitcraze/crazyflie-demos.git
cd crazyflie-demos/demos/scripts/cflib/multiranger/multiranger_wall_following
echo "$CFLIB_URI"
python3 multiranger_wall_following.py

Before running the demo, confirm that the Crazyflie, Crazyradio, Flow deck, and Multi-ranger deck are connected correctly, and verify that the terminal prints the complete radio URI assigned to the group. The official script reads CFLIB_URI through uri_helper.uri_from_env(); do not edit its default address. For the first test, use a low-speed, open, safe area and confirm hover and emergency-stop behavior first.

As shown in the figure below, lines 75 to 77 in the multiranger_wall_following.py file

In the process of initializing the instance, the first parameter angle_value_buffer controls the tolerance of the UAV's judgment angle. The larger the value, the slower the UAV's response will be, but the path will be relatively smooth. The smaller the value, the more agile the UAV's response will be and the path will jitter more. Reference_distance_from_wall This value controls the shortest distance the drone expects to be from the wall, in m. The larger the value, the farther the drone is from the wall. The value max_forward_speed represents the fastest forward speed of the drone, in m/s. Students can slightly modify these parameters and observe whether there are significant changes in the flight path of the drone. However, during this process, they must pay attention to the rationality of the parameters, and the values ​​should not be too extreme to ensure flight safety.

Experiment 2: More options for obstacle avoidance flight

Now that everyone is proficient in using optical flow sensors and obstacle avoidance sensors, let the drone take off from area A according to the obstacle map environment shown in the figure below. During the entire process, the flying height of the drone should not exceed 0.4m. Explore the entire map, that is, fly all the way without hitting the wall until it lands in area B. Now that the students have mastered the programming methods of motioncommander and multiranger, think about what kind of programming method can be used in such a map environment to logically complete the fixed flight route task in this map environment known in advance more efficiently.

Note that the content of this experiment will be related to part of the final assessment. This experiment hopes to improve and test the students' programming proficiency. If more clever code logic is used to implement such tasks, it will make everyone's practical assessment results even better.

Experiment 3: Manually control flight mapping

Official demo repository: Multiranger Point Cloud

multiranger_pointcloud.py comes from Bitcraze's official crazyflie-demos repository. Use the file in the official repository as the source of record; this manual lists the source link and classroom usage steps.

  • Repository: https://github.com/bitcraze/crazyflie-demos/tree/main
  • Demo directory: demos/scripts/cflib/multiranger/multiranger_pointcloud
  • multiranger_pointcloud.py: official file link

Recommended usage in the virtual machine:

cd ~/workspace
git clone https://github.com/bitcraze/crazyflie-demos.git
cd crazyflie-demos/demos/scripts/cflib/multiranger/multiranger_pointcloud
echo "$CFLIB_URI"
python3 multiranger_pointcloud.py

Before running the demo, confirm that the Crazyflie, Crazyradio, Flow deck, and Multi-ranger deck are connected correctly, and verify that the terminal prints the complete URI assigned to the group. The official script reads CFLIB_URI through uri_helper.uri_from_env(). This example requires a graphical desktop and Python visualization dependencies; if dependencies are missing, install numpy, vispy, and PyQt6 according to the official README. Run the first test in a safe area and confirm that the drone lands normally after the visualization window is closed.

After running this program, you will use the keyboard to manually control the drone to fly in an obstacle environment and sense the surrounding environment. Use the direction keys to control the flight direction of the drone. The drone will rise to the default height of 0.3m. The a and d keys are used to control the drone's steering slowly, the z and x keys are used to control the drone's steering quickly, and the height is controlled by the w and s keys. Since the multiranger's sampling speed is limited, if the drone flies too fast, the sampled environment map may not be clear. Therefore, if you want to get a clearer point cloud map, you can slightly slow down the drone's flight speed to get a clearer point cloud map.

Now run this program to construct a clear point cloud map of the map obstacles for this lesson.

Mapping Result Reference

When reviewing the mapping experiment, compare the physical arena structure with the point-cloud output, focusing on the outer boundary, internal baffles, and abnormal scattered points.
