Experiment 12: PositionCommander

1. Experimental goals

This lesson aims to enable students to master the core usage of the PositionHlCommander class and achieve precise path planning for drones.

  1. Understand the concept of spatial coordinate system

Understand the difference between the world coordinate system (absolute coordinates) and the body coordinate system (relative coordinates) during drone flight.

Master the origin (0,0,0) setting logic of Crazyflie combined with Flow deck v2 when powered on.

  1. Master the basic usage of the PositionHlCommander class

Learn to use take_off() and land() to implement automatic take-off and landing, and understand the role of default parameters (height, speed).

Core skills: Master the go_to(x, y, z) command to fly the drone to a specific absolute coordinate point in space.

Advanced skills: Master relative position control instructions such as forward(), back(), left(), right(), etc., and understand their mixed use with absolute coordinate control.

  1. Write a combination flight routine

You can use the above instructions comprehensively to write Python scripts to drive the drone to fly in specified geometric shapes (such as squares, rectangles or three-dimensional step trajectories).

2. Experimental preparation

The virtual machine system of the client connected to the drone has been configured previously. And you need to have the Linux system operating experience accumulated in the previous experiments. A little more basic python programming is needed.

Hardware equipment needed in this experiment:

  • Crazyflie 2.0
  • Crazyradio PA
  • Flow deck
  • Multiranger deck

First, let’s get to know the member functions and usage of the PositionHlCommander class:

The source code of this class can be viewed in the position_hl_commander.py file attached to this course as a reference.

Before writing the flight code, we need to "get to know" our commander - PositionHlCommander. This class encapsulates a series of instructions that allow us to tell the drone where to go just like directing traffic.

Here are the core functions we'll use frequently in this lesson:

1. Initialization and take-off: PositionHlCommander (constructor)

We usually use this class with the with statement, so that we can achieve "automatic takeoff and automatic landing", which is very safe and convenient.

Code example

Key parameters

x, y, z (floating point number): Set the starting coordinates of the drone. Usually keep the default value of 0.0, 0.0, 0.0, that is, take the starting position of the drone as the origin (important!!!).

default_height (floating point number): Default flight height (unit: meters). The drone will automatically hover at this height after taking off. The recommended setting is 0.3 to 0.5 meters.

default_velocity (floating point number): Default flight speed (unit: meters/second). It is recommended to set it to 0.3 for beginners and increase it to 0.5 for those who are proficient.

default_landing_height (floating point number): Landing height. This parameter needs to be adjusted if you want to take off on the table but land on the floor, or vice versa. Defaults to 0.0 (ground).

2. Absolute coordinate flight: go_to(x, y, z, velocity)

This is the core function to implement path planning. You tell the drone a specific coordinate point (X, Y, Z), and it will fly straight there.

Function prototype: pc.go_to(x, y, z=None, velocity=None)

Parameter details

x, y (required): X and Y coordinates of the target point (meters).

z (optional): target height. If left blank, keep the default height.

velocity (optional): The flight speed for this section of the journey. If left blank, the default speed is used.

Tip

This function is blocking, which means that the code will not continue until the drone flies to its destination.

The UAV always keeps its nose facing the positive direction of the X-axis (Yaw = 0) during flight and will not turn its head.

3. Relative coordinate movement (axial movement)

In order to simplify programming, PositionHlCommander provides a set of "front, back, left, and right" shortcut instructions.

Note: The "front, back, left, and right" here are relative to the coordinate axis, not relative to the drone's nose (because the nose is always facing the positive X-axis).

forward(distance) / back(distance)

Function: Move the specified distance along the X-axis in the positive direction (front) or negative direction (back).

Mathematical meaning: = + distance, Y and Z are unchanged.

left(distance) / right(distance)

Function: Move the specified distance along the Y-axis in the positive direction (left) or negative direction (right).

Mathematical meaning: = + distance, X and Z are unchanged.

Important: Crazyflie follows a right-handed coordinate system. When facing the positive X-axis, the positive Y-axis is on the left-hand side.

up(distance) / down(distance)

Function: Vertically rise or fall a specified distance along the Z axis.

Important notes

The initial position of the drone called by the positioncommander class is the position when the drone is powered on.

That is, every time you run the code with the positioncommander class, the drone must be placed at the take-off position, and then powered on after it is stable. At this time, the coordinates are positioned at (0, 0, 0), and then run the program. And because the positioning will deviate every time you fly and land, you must strictly follow this process every time you fly, and power on the drone stationary at the take-off position. That is, if the drone lands normally and smoothly, you must power on again and perform this step before running the next program. Otherwise, the positioning of the drone will be disordered and the drone will explode when taking off. Be careful!

Experiment 1: From square to cube

The following example program is given. The effect achieved by this program is that the drone takes off. On the plane with the z-axis coordinate of 0.5, it traverses four coordinate points in sequence, and the flight trajectory is a square. You can name the python script position_commander_demo.py

import time
import cflib.crtp
from cflib.crazyflie import Crazyflie
from cflib.crazyflie.syncCrazyflie import SyncCrazyflie
from cflib.positioning.position_hl_commander import PositionHlCommander
from cflib.utils import uri_helper
# URI to the Crazyflie to connect to
uri = uri_helper.uri_from_env(default='radio://0/80/2M/E7E7E7E7E4')
def simple_sequence():
with SyncCrazyflie(uri, cf=Crazyflie(rw_cache='./cache')) as scf:
# Arm the Crazyflie
scf.cf.platform.send_arming_request(True)
time.sleep(2.0)
with PositionHlCommander(
scf,
x=0.0, y=0.0, z=0.0,
default_velocity=0.25,
default_height=0.35,
controller=PositionHlCommander.CONTROLLER_PID) as pc:
pc.go_to(1.0, 0.0, 0.5)
pc.go_to(1.0, 1.0, 0.5)
pc.go_to(0.0, 1.0, 0.5)
pc.go_to(0.0, 0.0, 0.5)
if __name__ == '__main__':
cflib.crtp.init_drivers()
simple_sequence()

Now, modify the program so that the drone's flight trajectory traverses eight points of a three-dimensional rectangle in the air. Note that the height of the rectangle is 0.5m, that is, set the maximum height of the z-axis to 1m.
