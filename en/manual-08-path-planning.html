Experiment 8: Path-planning Simulation

1. Experimental goals

Learn how to transfer files from a physical host to a virtual machine

Successfully run and compile a project using ROS.

Understand the role of path planning by running several classic path planning algorithms.

2. Experimental preparation

Experiment 2 has configured the Ubuntu20.04 virtual machine of ROS.

3. Experimental steps

Experiment 1: Transfer the code project to be run to the virtual machine and decompress it

First, we create a folder named workspace in the user directory of the virtual machine.

Click the user directory icon in the upper left corner, right-click on the blank space, select new folder, and name the new folder workspace.

Click the green button create in the upper right corner to create

Then click the icon of the created workspace folder to enter the folder.

At the same time, open the project archive in the Course Materials Package: course-materials/00_project_archives/uav_motion_planning.zip. Select this archive on the host machine and drag it directly into workspace in the virtual machine.

Then right-click and select extract here to extract the file into the directory.

Next, we click on the icon of the decompressed project file. After entering the interface, right-click on the blank space and select open in terminal to open a terminal that has entered the current file directory.

As shown in the figure, the blue text in the terminal represents the location of the file directory where the terminal is currently located. It can be seen that in Linux, we can not only access the folder using the above graphical operation method, but also use the command line terminal.

As shown in the figure below, we can enter the pwd command in the terminal to view the path of the file directory where the terminal is currently located, that is, which folder the terminal is accessing. Enter the ls command again to see what files are in the current folder directory. These are the two ways to access files graphically and from the command line in the Linux system. You can see that the number and names of files displayed in the graphical interface and the terminal are the same, because both are accessing the same folder.

Experiment 2: Configure project running environment and compile code

After entering the directory, first install the necessary dependencies to run the program, and enter the following command in the terminal (you may be asked to enter the user password to obtain permissions):

sudo apt install libeigen3-dev

Then enter: sudo apt-get install ros-noetic-octomap-ros

Next we will compile the relevant dependencies of the project and enter the following command in the directory of the current terminal:

cd 3rd/osqp/build

The cd command is a commonly used command under Linux to enter the next-level directory. The command entered here means entering the 3rd/osqp/build directory under the /workspace/uav_motion_planning directory.

Similarly, we can also click to enter the file directory using the graphical interface. This step is just to let everyone understand the logic of file directory access under Linux.

Next enter the following command (if system dependencies are missing or the software source is unavailable, check the network or offline-source configuration first):

cmake ..

cmake is a commonly used cross-platform programming language compilation tool.

Next, enter the make command and compile to 100%, which is considered successful.

Next enter the following command:

sudo make install

Now that the compilation of the osqp module part is completed, the compilation of the osqp-eigen module remains.

Enter the following command:

cd ..

You can return to the upper directory. Here we enter three times in the previous terminal until we return to the main directory of the project.

Here we enter the following command:

cd 3rd/osqp-eigen/build

Enter the directory of another module that needs to be compiled.

Enter the cmake .. command in the same process.

Then enter the make command for further compilation.

Then enter the sudo make install command to complete the last step of the module.

Then we return to the main directory of the project file again.

For the final step of compilation, enter:

catkin_make -DCMAKE_CXX_STANDARD=14

Wait for the compilation to complete. All preparations have been completed and you can run the program.

Experiment 3: Run the classic path planner

Right-click the blank space of the current terminal and select new window to quickly create a new terminal in the directory.

Enter the following commands in both terminals:

source devel/setup.bash

This command is used to import relevant environment variables before running the ros program each time, so that ros can find the program you want to run.

1. A_star algorithm

Enter the following command in the first terminal:

roslaunch plan_manage single_run_in_sim.launch

Enter the second terminal:

roslaunch test test_astar_searching.launch

You can see the map environment in the simulated rviz interface.

Use the mouse wheel or computer touchpad to zoom in and out to see the entire environment. At this time, click the 3D Nav Goal option in the menu bar above, click on any point on the map to post an end point information to the drone, and then you can view the red path for the drone to reach the end point calculated by the A_star path planning algorithm.

2. RRT algorithm

For the two terminals running above, click on the terminal respectively, use the ctrl plus c command to interrupt the running program, and then perform the following steps:

Enter the following command in the first terminal:

roslaunch plan_manage single_run_in_sim.launch

Enter the second terminal:

roslaunch test test_rrt_searching.launch

You can see the map environment in the simulated rviz interface.

Publish the end point in the same way, and you can see the effect of path planning by the rrt algorithm.

3. RRT_star algorithm

In the same process, for the two terminals running above, click the mouse on the terminal respectively, use the ctrl plus c command to interrupt the running program, and then perform the following steps:

Enter the following command in the first terminal:

roslaunch plan_manage single_run_in_sim.launch

Enter the second terminal:

roslaunch test test_rrt_star_searching.launch

You can see the map environment in the simulated rviz interface.

Release the end point in the same way, and you can see the effect of path planning by the rrt_star algorithm. Observe and compare the difference between the rrt algorithm and its improved rrt_star algorithm. What are the advantages of the improved algorithm?
