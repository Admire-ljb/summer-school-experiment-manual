实验 1: 安装配置虚拟机

Virtual Machine Installation and Configuration

实验手册1——安装配置虚拟机

一、实验目标

掌握VMware虚拟化软件的基础安装与配置方法。

学会在VMware中创建Ubuntu 20.04 LTS虚拟机，并完成系统安装。

理解虚拟机的核心概念（如硬件隔离、快照管理）及其在开发测试中的应用场景。

二、实验准备

硬件要求

物理机配置：建议8GB以上内存、50GB以上可用磁盘空间（虚拟机需占用约20GB）。

软件资源（可从官网下载，为节省时间由助教拷贝）

VMware Workstation Pro 16

Ubuntu 20.04 LTS Desktop版镜像文件

工具准备

任务管理器（查看物理机CPU核心数）。

文本编辑器（如记事本，用于记录虚拟机配置参数）。

三、实验步骤

阶段1：VMware软件安装

具体图文流程在浏览器中打开此链接：

下载与安装

访问VMware官网，下载Workstation Pro 16安装包。

双击安装包，按向导提示完成安装（建议安装路径选择D盘）。

安装完成后，创建桌面快捷方式。

验证安装

启动VMware，检查主界面是否显示“创建新虚拟机”按钮。

命名与存储

虚拟机名称：Ubuntu20.04_Test。

存储路径：选择D盘下新建的VMware_Ubuntu文件夹。

硬件配置

处理器：根据物理机核心数分配（如4核CPU分配2核，每个核心1-2线程）。

内存：

基础任务：2GB

多任务/开发环境：4GB

保留至少4GB内存给物理机。

磁盘：

类型：SCSI（推荐）

容量：分配60GB，选择“存储为单个文件”。

网络：默认NAT模式（自动获取IP，可访问外网）。

加载ISO镜像

在虚拟机设置中，选择CD/DVD驱动器，加载下载的Ubuntu 20.04 ISO文件。

阶段3：安装Ubuntu 20.04

启动虚拟机

点击“开启此虚拟机”，等待Ubuntu安装引导界面加载。

系统安装流程

语言选择：中文（简体）。

键盘布局：默认English（US）。

安装类型：选择“最小安装”（节省磁盘空间）或“正常安装”（含常用软件）。

分区设置：

选择“其他选项”，手动创建分区表。

分区方案：

/（根目录）：20GB，EXT4文件系统。

/home（用户目录）：剩余空间，EXT4文件系统。

Swap（交换分区）：2GB（内存不足时备用）。

用户信息：设置用户名（如student）、密码（需牢记）。

安装完成：点击“重启”，移除ISO镜像后按Enter键。

系统初始化

登录后，根据提示跳过在线账号绑定和隐私设置。

调整任务栏图标：右键隐藏不常用应用，添加“终端”到收藏夹。

五、实验总结与拓展

核心知识点回顾

虚拟机通过软件模拟硬件，实现操作系统隔离。

快照功能可快速保存/恢复系统状态，适用于开发测试场景。

拓展任务（选做）

在虚拟机中安装Python3.10版本（可借助互联网）。
